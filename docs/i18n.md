# Internationalization (i18n)

## Stack

- `i18next` + `react-i18next` — core library
- `i18next-browser-languagedetector` — automatic browser language detection
- `i18next-http-backend` — loading translations from the server
- `i18next-localstorage-backend` — caching translations in localStorage
- `i18next-chained-backend` — backend chain: localStorage first, then HTTP

## Languages

| Code | Language | Status |
|---|---|---|
| `uk` | Ukrainian | default |
| `en` | English | — |

The language is stored in localStorage and automatically detected on the first visit.

## Namespaces

Translations are split into namespaces for code-splitting:

| Namespace | Purpose |
|---|---|
| `translation` | General translations (navigation, footer, common elements) |
| `auth` | Authentication (login, registration, password recovery) |
| `cart` | Cart and checkout |
| `catalog` | Catalog and product cards |
| `article` | Articles / blog |
| `tags` | Tags and labels |
| `favourites` | Favourite products |
| `search-result` | Search results |

## Translation Files

Location: `public/locales/{{lng}}/{{ns}}.json`

```
public/locales/
├── uk/
│   ├── translation.json
│   ├── auth.json
│   ├── cart.json
│   ├── catalog.json
│   ├── article.json
│   ├── tags.json
│   ├── favourites.json
│   └── search-result.json
└── en/
    └── ... (same structure)
```

## Usage

```tsx
import { useTranslation } from 'react-i18next';

const MyComponent = () => {
  const { t } = useTranslation('catalog');
  return <h1>{t('page_title')}</h1>;
};
```

To access a namespace other than `translation`, pass its name to `useTranslation()`.

## Adding a New Translation

1. Add the key to the corresponding JSON file for both languages (`uk` and `en`)
2. Use `t('key')` in the component with the correct namespace

## Adding a New Namespace

1. Create `public/locales/uk/<namespace>.json` and `public/locales/en/<namespace>.json`
2. Add the namespace to the i18n configuration in `src/shared/config/i18n/`
3. Use it: `useTranslation('<namespace>')`
